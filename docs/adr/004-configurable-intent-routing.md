# ADR-004：可配置意图路由

- 状态：已采纳

## 背景

“最新文章”“热门文章”和标签查询依赖实时结构化数据，不适合仅靠 RAG 文本片段回答。硬编码意图又难以扩展和让管理员调整。

## 决策

使用持久化 IntentRoute、正则优先/LLM 兜底的检测器，以及可组合 PipelineProcessor。命中后跳过 RAG，读取公开 Post，按步骤过滤排序，再由 LLM 组织回答。

## 后果

- 管理员可以修改触发词、优先级、步骤和输出模板。
- 处理器、校验、Console、Trace、测试和文档必须同步演进。
- LLM 分类失败或超时时回到 RAG。
- 处理器失败默认清空候选，显式 `onFailure=keep` 才保留上一步结果。
