# AI 摘要

> 适用读者：Halo 作者、内容运营人员

AI 摘要通过 Halo 摘要扩展点或后台批量操作生成文章摘要。

![AI 摘要管理](../../assets/readme/screenshots/console-excerpt.jpg)

## 两种使用方式

### 编辑器自动摘要

开启功能后，在 Halo 文章编辑页选择自动生成摘要并保存。插件读取文章内容，调用模型生成摘要。

勾选“自动生成摘要”后，保存或发布文章时 Halo 会调用 AI 智能套件的摘要生成器：

1. 插件读取当前文章内容，并按“最大输入字符数”截取送入模型的正文。
2. 使用 AI 摘要页面配置的长度、温度、输出 Token 和 Prompt 生成摘要。
3. 生成结果写入文章的摘要字段，供文章列表、主题页面和 SEO 信息使用。

如果 Chat 模型暂时不可用，插件会自动退回到正文纯文本截取，不会因为摘要生成失败而阻止文章保存。AI 摘要功能关闭时，勾选该选项也只会生成截取摘要，不会调用模型。

如果已经手工填写并希望长期保留摘要，应取消勾选“自动生成摘要”，否则后续保存文章时可能再次覆盖摘要内容。

### 后台管理

摘要页面支持查看文章摘要状态、单篇生成、清除、批量生成和批量清除。

## 配置建议

| 配置 | 建议 |
| --- | --- |
| 目标长度 | 120～200 个中文字符 |
| 输入上限 | 覆盖文章核心部分，同时控制 token |
| 温度 | 0.2～0.4 |
| 输出上限 | 为目标长度留出余量 |
| 自定义 Prompt | 强调事实、禁止评价和虚构 |

## 安全边界

- 生成摘要会覆盖目标文章的摘要字段，批量操作前先抽样验证。
- 清除摘要不会删除文章正文。
- 输入超过上限时只会把限定长度的内容送入模型，长文尾部信息可能缺失。

## 验证

- 摘要只包含正文中存在的事实。
- 人名、数字和结论与原文一致。
- 文章列表和 SEO 展示没有出现截断异常。
- 自动摘要关闭后，文章保存不会产生模型调用。

完整默认值见 [配置参考](../reference/configuration-reference.md)。
