# AI 搜索

> 适用读者：Halo 站长、主题集成人员

AI 搜索在关键词结果之外生成一段基于站内文章的综合回答，同时保留传统结果供访客继续浏览。

![AI 搜索配置与预览](../../assets/readme/screenshots/console-search.jpg)

## 数据路径

- 关键词结果：`GET /search/halo-results`，直接读取插件 Lucene 索引。
- AI 综合回答：`POST /search/answer`，进入独立搜索 RAG 流程并使用 SSE。
- 搜索回答不会进入自定义意图路由。

## 入口依赖说明

AI 搜索的检索能力不依赖 Halo 官方搜索插件：关键词结果来自本插件自己的 Lucene 索引，AI 综合回答也由本插件接口生成。

但页面上是否存在可点击的“搜索入口”，取决于当前主题、官方搜索插件或站点自定义入口。插件会接管主题暴露的 `window.SearchWidget.open()`；如果站点没有搜索按钮，访客仍可通过 `Ctrl/Cmd + K` 或 `/` 快捷键唤起 AI 搜索，也可以由主题或自定义按钮主动调用 `window.SearchWidget.open()`。

## 配置步骤

1. 确保文章索引正常。
2. 确认主题、官方搜索插件或自定义代码提供了搜索入口；没有入口时至少验证快捷键可唤起。
3. 开启搜索功能。
4. 按需开启 AI 综合回答。
5. 设置关键词结果数量、专用 System Prompt 和输出上限。
6. 独立设置搜索主题与主题色。
7. 在实时预览中验证关键词结果和回答区域。

## 提示词建议

搜索回答应短于聊天回答，直接总结结果差异，并避免生成索引中不存在的文章。专用提示词为空时会回退到对话提示词。

## 常见问题

| 现象 | 检查 |
| --- | --- |
| 关键词结果为空 | 索引、搜索总开关、文章是否公开 |
| 有关键词结果但无 AI 回答 | `showAiAnswer`、Chat 模型、SSE 代理 |
| 页面上没有搜索按钮 | 主题或官方搜索插件是否提供入口；也可使用 `Ctrl/Cmd + K`、`/` 或自定义按钮调用 `window.SearchWidget.open()` |
| 搜索入口没有接管主题 | 主题是否暴露 `window.SearchWidget.open()`、Widget 注入和浏览器控制台 |
| 回答最后一次性出现 | Nginx/CDN 缓冲 |
| 结果摘要含危险 HTML | 前端应保持当前清洗逻辑，不直接渲染未知 HTML |

## 验证

- 使用文章标题中的精确词测试关键词结果。
- 使用语义问题测试 AI 综合回答。
- 确认传统结果和 AI 引用指向同一站点文章。
- 点击主题搜索入口，或使用 `Ctrl/Cmd + K`、`/` 快捷键测试弹框。

相关接口见 [Public API](../api/public-api.md)。
