# 运营智能体

> 适用读者：内容运营人员、站长

运营智能体读取近期问答日志与公开文章，识别访客需求没有被现有内容覆盖的部分，并输出选题、提纲或旧文优化建议。

![运营智能体](../../assets/readme/screenshots/console-agent.jpg)

## 当前边界

当前版本是只读分析：不会自动创建、修改或发布文章。每次运行保存为 AgentTaskRecord，刷新页面后仍可查看历史报告。

## 运行参数

- `days`：分析最近多少天的问答。
- `maxLogs`：最多读取多少条日志。
- `maxTokens`：报告输出预算。

默认值分别为 30、80、5000。

## 使用流程

1. 先积累足够的真实问答记录。
2. 选择分析时间范围和日志量。
3. 提交任务，页面展示后台进度。
4. 查看内容缺口优先级、证据问题和相关文章。
5. 人工判断建议是否值得进入选题计划。

## 如何判断建议质量

- 是否由多条真实问题支持，而不是单次偶然提问。
- 站内是否确实缺少对应内容。
- 建议是否能链接到现有文章进行更新，而非一律新建。
- 是否存在隐私、敏感问题或明显垃圾请求。

## 任务记录

任务记录支持列表、详情和删除。删除只影响分析报告，不会删除问答日志或文章。

相关接口见 [Console API](../api/console-api.md)。
